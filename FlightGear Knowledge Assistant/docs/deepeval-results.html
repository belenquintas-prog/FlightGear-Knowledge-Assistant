FlightGear supports multiple flight dynamics models (FDMs) which can be configured during the use of the simulator. The main supported models are:

2. LaRCsim: The original flight dynamics model used in FlightGear until 2002, developed by NASA. It had limitations since much was hardcoded instead of configured via files.

4. UIUC model: Developed by a University of Illinois at Urbana-Champaign team, this model incorporates nonlinear aerodynamics modeling for realistic behaviors in extreme flight conditions (e.g., stall and limit angles of attack). It includes specialized aircraft models like the Airwave Xtreme 150 hang glider and the Wright Brothers 1903 plane.

5. Magic Carpet (Magic): A special-purpose simplified flight model useful for scenario creators who want to move in the virtual world without physics concerns. It has been replaced by the "ufo" model.

6. Network model: Opens a UDP network protocol interface to receive flight dynamics data from an external program and feed it into FlightGear.

It is also possible to connect FlightGear to an external FDM running locally via a pipe, though this is not recommended for beginners.

Configuration / Limits:
- The flight dynamics model can be configured at runtime in the simulator.
- Each model offers different levels of realism and complexity.
- The network FDM uses a UDP interface with configurable host and port parameters.
- The Magic Carpet model is simplified and used mainly for convenience in scenarios.
- The UIUC model focuses on nonlinear aerodynamics and extreme flight conditions.
- The LaRCsim model is legacy and has limitations compared to newer models.

Actual Output:

Explicación Funcional:
La variable de entorno FG_ROOT se utiliza para indicar a FlightGear la ubicación del paquete base de datos (FG-Data). Esto puede ser necesario cuando no se usa el lanzador integrado o cuando FlightGear no puede encontrar automáticamente sus archivos de datos. En lugar de pasar la opción --fg-root al ejecutar fgfs, se puede configurar esta variable de entorno para que FlightGear sepa dónde buscar los datos base.

Configuración en Windows:
Aunque la documentación recuperada da ejemplos principalmente para terminales estilo Bourne y C (Linux/macOS), la configuración en Windows debe realizarse mediante la configuración de variables de entorno del sistema o del usuario. En Windows, esto usualmente se hace así:
1. Abrir "Configuración avanzada del sistema".
2. Ir a "Variables de entorno".
3. Crear una nueva variable (o editar si ya existe) con nombre FG_ROOT.
4. Asignar como valor la ruta del directorio donde están los archivos FG-Data.
Por ejemplo, si FG-Data está en C:\Program Files\FlightGear\data, ese debe ser el valor de FG_ROOT.

No obstante, el documento no detalla explícitamente pasos para Windows, sólo menciona que configurar variables de entorno depende del sistema operativo y shell utilizado.

Configuración / Límites:
